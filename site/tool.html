What files does the tool expect?

Standard CSV exports from most HRIS platforms work without modification. The engine handles column mapping automatically.

Column Required Notes
worker_id Required Employee ID. Used for Tier 1 exact matching.
full_name Required Used in name similarity matching. Normalized internally.
date_of_birth Recommended Last-4 + DOB matching (Tier 2). YYYY-MM-DD preferred.
hire_date Recommended Mismatch detection and root cause analysis. YYYY-MM-DD.
salary Optional Annual base salary. Used in mismatch categorization and correction output.
worker_status Optional Active / Inactive / Leave. Used in status mismatch detection.
position / job_title Optional Job/org mismatch pack and correction output.
last4_ssn / last4_ni Optional Last-4 identifier for Tier 2 matching. Never stored or exported after matching.

Column names don't have to match exactly. The mapping layer handles common variations. See the How To guide for details.
