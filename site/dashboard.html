Expected CSV columns

Standard HRIS exports work without modification. The engine maps columns automatically.

ColumnRequiredNotes
worker_idRequiredEmployee ID. Used for Tier 1 exact matching.
full_nameRequiredUsed in name similarity matching. Normalized internally.
date_of_birthRecommendedLast-4 + DOB matching (Tier 2). YYYY-MM-DD preferred.

Terms & Definitions

Quick reference for key concepts in reconciliation and audit outputs.

TermWhat It Means
Match Source How the system paired the two records. worker_id = exact ID match (highest confidence). dob_name = matched by name + date of birth (fuzzy).
Confidence How certain the system is in the match (0 to 1.0). 1.0 = exact ID match. Scores below 0.85 are flagged for review.
Priority Score Urgency score for manual review. 70+ = critical (review before import). 40-69 = high. 20-39 = medium. 0-19 = low / auto-approve candidate.
Safe No issues detected. This record passed all automated checks and is approved for import.
Needs Review One or more fields require manual verification before the record can be imported. See the Review_Queue workbook sheet.
Wrong Match The engine could not confirm this pairing is the same person. Verify identity before applying any corrections. See the Held_Corrections sheet.
Held Blocked from automated processing. A manual decision is required before this record can proceed.
Fix Types Which fields changed between systems: Salary, Status, Hire Date, Job / Org. A record can have multiple fix types.
Sanity Gate An automated health check that runs before corrections are generated. FAIL means something unusual was detected (e.g. too many active/$0 salary records) and corrections are blocked until resolved.
