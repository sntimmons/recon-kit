About

What is Data Whisperer?

The tool built to solve the HRIS migration problem that every consultant knows and nobody talks about.

The problem

Every HRIS migration has the same painful moment: somewhere between exporting data out of the legacy system and importing it into the new one, records get corrupted. Hire dates reset to a default. Salaries truncate. Status fields get dropped. Worker IDs shift format. The data that arrives in Workday or SuccessFactors or Oracle HCM is not the same data that left the old system. Nobody catches it until payroll runs.

The standard approach is manual spot-checks and VLOOKUP spreadsheets. On a 5,000-employee migration, that means days of analyst time, subjective sampling, and a best-guess sign-off. It's not rigorous, and when something slips through, it becomes a payroll correction, or worse, a compliance issue.

What Data Whisperer does

Data Whisperer is a deterministic reconciliation engine. It takes your legacy system export and your new system import, matches every employee across both datasets using a tiered identity-resolution algorithm, and then audits every matched pair for field-level discrepancies.

The matching engine uses six strategies in priority order:

  • Exact worker ID match (highest confidence)
  • Reconciliation ID or primary key match
  • Last-4 digits of SSN/NI number + date of birth
  • Date of birth + normalized full name
  • Full name + hire date
  • Name similarity scoring (fallback)

Each match is assigned a confidence score and a strategy label. Conflicts are resolved deterministically via staged 1-to-1 matching. No probabilistic guessing. If the engine can't resolve a match with confidence, it routes the record to a human review queue.

Once matched, every pair is audited across five field categories: salary, pay rate, worker status, hire date, and job/org data. Mismatches are quantified, bucketed, and evaluated against configurable policy thresholds. If the data is clean enough to proceed, the engine auto-generates Workday-ready correction files for each field category. If it isn't, the sanity gate blocks corrections and flags the migration for review.

Who it's built for

Data Whisperer is built for the people who actually do HRIS migrations, not the project managers who oversee them. That means:

  • HRIS consultants running Workday, Oracle, SAP, or SuccessFactors implementations
  • Enterprise HR operations teams managing data quality during a system switch
  • HR data analysts who've been handed a 40,000-row CSV and told to "make sure it's right"
  • Implementation teams who need to evidence data accuracy for a go-live sign-off

If you've ever spent a week building a reconciliation spreadsheet from scratch for a migration you've already done three times before, this tool was built for you.

What it doesn't do

Data Whisperer is not an ETL tool. It doesn't transform or migrate data. It doesn't connect to your HRIS via API. It doesn't run in the cloud or share your data with anyone. It takes two CSV files, runs locally on your machine, and tells you exactly where the discrepancies are before you go live. That's the whole job.
